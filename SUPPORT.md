# Support

## What To Use Issues For

Please use GitHub issues for:

- reproducibility failures
- raw artifact inconsistencies
- release-audit regressions
- benchmark or verifier bugs
- documentation errors that change the supported claim surface

## What Not To Use Issues For

This repository is not positioned as a general AI product or support channel.

Please do not use issues for:

- requests to broaden the public claim beyond the current evidence
- requests for general chatbot features
- requests to treat the repo as a general theorem prover
- requests that depend on private credentials or unpublished artifacts

## Evidence Rule

When reporting a problem, include:

- the exact commit or tag
- the exact command you ran
- the raw JSON artifact that shows the discrepancy
- the relevant environment detail if it affects live-model execution

Raw artifacts outrank Markdown summaries.
